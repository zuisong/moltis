Get Started

Install Moltis and start your AI assistant in minutes

~ quickstart

$ curl -fsSL https://www.moltis.org/install.sh | sh

$ brew install moltis-org/tap/moltis

# Multi-arch: amd64/arm64

$ docker run -d \

--name moltis -p 13131:13131 -p 1455:1455 \

-v moltis-config:/home/moltis/.config/moltis \

-v moltis-data:/home/moltis/.moltis \

-v /var/run/docker.sock:/var/run/docker.sock \

ghcr.io/moltis-org/moltis:latest

# Open http://localhost:13131 and complete the setup

# Build from source

$ cargo install moltis --git https://github.com/moltis-org/moltis

# Download packages from GitHub Releases

# Debian/Ubuntu (.deb)

$ sudo dpkg -i moltis_*_amd64.deb

# Fedora/RHEL (.rpm)

$ sudo dnf install moltis-*.x86_64.rpm

# Arch Linux (.pkg.tar.zst)

$ sudo pacman -U moltis-*-x86_64.pkg.tar.zst

# Snap

$ sudo snap install moltis

# AppImage

$ ./moltis-*-x86_64.AppImage

Design com segurança em primeiro lugar

Um assistente IA em que você pode confiar na sua máquina

Por que você deveria confiar nele?

Moltis é um arquivo. Você baixa, executa, pronto. Não há loja de plugins, gerenciador de pacotes ou runtime que possa ser comprometido. A aplicação inteira é um único binário que você pode verificar.

Sua IA roda em sandbox por padrão. Ela não pode ler seus arquivos, executar comandos ou navegar na web a menos que você permita explicitamente. Cada ação de ferramenta pode exigir sua aprovação — ou você pode pré-aprovar as que confia.

Suas senhas e chaves API são encapsuladas para nunca serem registradas acidentalmente, e a memória é zerada no momento em que não são mais necessárias. A linguagem em que é construído (Rust) torna classes inteiras de bugs de segurança impossíveis no nível do compilador.

O que você não encontrará aqui

Sem marketplace de plugins para ser comprometido. Sem runtime JavaScript. Sem árvore de dependências de 1.200 pacotes que você não consegue revisar. Um binário que você pode auditar de ponta a ponta.

Como protegemos você

E se... Como Moltis lida com isso
Alguém encontra sua instância Senha ou passkey obrigatória desde o primeiro início, HTTPS automático, limitação de taxa
IA tenta executar código perigoso Tudo roda em sandbox (Docker ou Apple Container). Cada ação precisa da sua aprovação primeiro
Uma dependência é comprometida Não há pacotes npm/pip para comprometer. Um binário, dependências auditáveis. Integrações como Telegram são código integrado e testado
IA acessa sua rede interna Endereços de rede internos e privados são bloqueados automaticamente
Alguém tenta roubar suas chaves Chaves API e senhas são ocultadas dos logs e zeradas na memória após o uso
Sessão é sequestrada Verificação de origem previne ataques cross-site, HTTPS habilitado por padrão
Acesso não autorizado Passkeys, chaves API com escopo, políticas de permissão por ferramenta

Execução em sandbox

Docker & Podman

Cada conversa tem seu próprio container isolado. Comandos shell e navegação web acontecem dentro do sandbox, não no seu sistema real.

Apple Containers

No Mac, usa a virtualização integrada da Apple para isolamento leve — sem necessidade de Docker.

Sandbox WASM

Sandbox baseado em Wasmtime com failover genérico. Isolamento leve sem containers.
